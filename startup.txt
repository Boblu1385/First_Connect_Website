ROB -- i seem to remember some sort of "build" operations? is that typical?

AI -- Yes, there IS a build operation, but **Netlify handles it automatically** for you!

Here's how it works:

**What happens when you push to `main`:**

1. You push your code: `git push origin main`
2. Netlify detects the push
3. **Netlify automatically runs:** `npm run build` (or whatever build command is configured)
4. This creates a dist folder with optimized production files (minified JS, CSS, etc.)
5. Netlify deploys the dist folder to your live site

**You don't need to manually build** because Netlify does it on their servers.

**If you wanted to test a production build locally:**
```bash
npm run build
npm run preview
```
- `npm run build` - Creates the optimized dist folder
- `npm run preview` - Serves the built files locally to test

**Key difference:**
- **Development:** `npm run dev` - Fast, unoptimized, hot-reload for coding
- **Production:** `npm run build` - Optimized, minified files for live deployment (Netlify does this automatically)

So your workflow remains the same - just push to `main` and Netlify handles the build. You can verify the build succeeded by checking the Netlify dashboard or waiting for the deployment confirmation.
